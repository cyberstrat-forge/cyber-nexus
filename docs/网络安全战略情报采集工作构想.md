# 📘 网络安全战略情报采集系统构想

---

# 1. 项目目标

## 1.1 总体目标

构建一个**独立、可扩展、可云化的战略情报采集系统**，实现：

- 多来源战略情报自动化采集
    
- 正文抽取与结构化处理
    
- 数据清洗与标准化
    
- 战略价值初筛
    
- 增量更新机制
    
- API 服务化输出
    
- 与 AI 情报分析系统解耦
    

形成公司战略情报体系中的：

> “数据生产引擎层”

---

## 1.2 设计原则

- 🔹 来源质量优先
    
- 🔹 全流程自动化
    
- 🔹 模块化解耦
    
- 🔹 可扩展
    
- 🔹 云原生兼容
    
- 🔹 面向战略分析，而非通用舆情
    

---

# 2. 系统总体架构

---

## 2.1 架构图

```text
┌──────────────────────────────────────┐
│               情报来源层              │
│ RSS / 网站 / YouTube / 播客 / 微信     │
└──────────────────┬───────────────────┘
                   ↓
┌──────────────────────────────────────┐
│            采集适配层（Connectors）   │
│  RSS采集 / 爬虫 / API适配 / 定时任务   │
└──────────────────┬───────────────────┘
                   ↓
┌──────────────────────────────────────┐
│            全文抽取层                  │
│        正文识别 / 去广告 / 去噪声       │
│            trafilatura                 │
└──────────────────┬───────────────────┘
                   ↓
┌──────────────────────────────────────┐
│            数据处理层                  │
│ 去重 / 清洗 / 标准化 / 初筛评分         │
│ 战略价值初评估                          │
└──────────────────┬───────────────────┘
                   ↓
┌──────────────────────────────────────┐
│            数据存储层                  │
│ PostgreSQL（主数据库）                 │
│ Markdown（交换与版本管理）              │
└──────────────────┬───────────────────┘
                   ↓
┌──────────────────────────────────────┐
│            API 服务层                  │
│ 新情报查询 / 增量拉取 / 状态更新         │
└──────────────────┬───────────────────┘
                   ↓
          AI 情报卡片处理应用
```

---

## 2.2 模块说明

### （1）情报来源层

支持：

- RSS/Atom
    
- 战略媒体网站
    
- YouTube 频道
    
- 播客
    
- 微信公众号（可通过转换或抓取）
    

---

### （2）采集适配层

负责：

- 调用不同来源接口
    
- 定时拉取
    
- 增量更新
    
- 统一数据格式输出
    

---

### （3）全文抽取层

核心作用：

- 提取正文内容
    
- 去除导航
    
- 去广告
    
- 保留结构
    
- 提高后续分析质量
    

推荐核心工具：

- trafilatura
    

---

### （4）数据处理层

包括：

- 去重（URL + 内容 Hash）
    
- 格式标准化
    
- 时间标准化
    
- 语言标识
    
- 战略初筛评分
    

初筛目的：

减少 AI 情报卡片系统的噪音负担。

---

### （5）存储层

双形态存储：

- PostgreSQL：结构化主数据
    
- Markdown：跨系统交换格式
    

Markdown 用于：

- 与情报分析应用衔接
    
- Git 管理
    
- 人工审阅
    
- 可迁移性
    

---

### （6）API 服务层

提供：

- 查询新情报
    
- 增量获取
    
- 按时间/评分筛选
    
- 标记已处理
    
- 支持分页
    

实现类似：

> “情报订阅接口”

---

# 3. 工作流程（Workflow）

---

## Step 1：情报源管理

- 建立来源清单
    
- 来源分级
    
- 质量评估
    
- 可用性监控
    
- 定期审查
    

原则：源质量决定系统质量。

---

## Step 2：采集

- 定时拉取
    
- 增量更新
    
- 支持多来源并行
    

---

## Step 3：全文抽取

- 自动正文识别
    
- 去广告
    
- 去噪
    
- 保留标题与结构
    

输出：

干净的正文内容。

---

## Step 4：去重与清洗

- URL 去重
    
- 内容 hash 去重
    
- 标准化字段
    
- 统一时间格式
    

---

## Step 5：战略初筛

基于：

- 关键词规则
    
- 简单语义匹配
    
- 来源权重
    
- 基础评分模型
    

输出：

战略相关性评分。

---

## Step 6：存储

- 写入数据库
    
- 生成 Markdown
    
- 更新状态
    

---

## Step 7：API 对外服务

- 提供新数据接口
    
- 支持 AI 系统订阅式访问
    

---

- Docker 化

- 支持本地/云混合架构

# 5. 与情报处理应用的对接说明

---

## 5.1 架构原则

采集系统与 AI 分析系统：

- 完全解耦
    
- 通过 API 通信
    
- 使用 Markdown 作为交换载体
    

---

## 5.2 数据流

1. 采集系统产生新情报
    
2. 写入数据库
    
3. 生成 Markdown
    
4. AI 系统通过 API 拉取
    
5. AI 系统生成情报卡片
    

---

## 5.3 对接优势

- 可独立升级
    
- 可迁移至云
    
- 不影响分析系统
    
- 支持未来事件驱动模式
