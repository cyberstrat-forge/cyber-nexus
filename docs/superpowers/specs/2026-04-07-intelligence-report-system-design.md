# 情报报告体系设计规范

> 本文档定义 market-radar 插件情报报告体系的设计规范，包括日报、周报、月报、年报的定位、结构和写作要求。

## 设计原则

### 层级聚合模型

报告体系采用**层级聚合**设计，高级别报告从低级别报告聚合生成，而非直接从情报卡片生成。

```
情报卡片
    ↓ 生成
日报（信号）
    ↓ 聚合 + 下钻
周报（主题）
    ↓ 聚合 + 下钻
月报（变化）
    ↓ 聚合 + 下钻
年报（趋势）
```

### 下钻设计

高级别报告支持下钻到低级别报告，但不一定需要直达情报卡片。

| 报告 | 主输入 | 按需下钻 | 是否访问情报卡片 |
|------|--------|----------|------------------|
| 日报 | 情报卡片 | - | ✅ 是 |
| 周报 | 日报集合 | 情报卡片 | ✅ 可选 |
| 月报 | 周报集合 | 日报 | ❌ 否 |
| 年报 | 月报 + 周报集合 | 日报 | ❌ 否 |

### 时间边界

| 报告 | 时间范围 |
|------|----------|
| 日报 | 当日（00:00-23:59） |
| 周报 | 周一至周日（ISO 周） |
| 月报 | 自然月（1日至月末） |
| 年报 | 自然年（1月1日至12月31日） |

## 日报设计

### 核心定位

**信号导向**：指导情报分析人员关注特定的情报。

### 受众

执行层（安全团队、情报分析师）

### 输入来源

情报卡片（按 `created_date` 过滤当日）

### 内容结构

```markdown
# 情报日报 | YYYY-MM-DD

## 执行摘要
{编辑点评视角，当日情报总体情况，3-5 句话}

## 情报概览

### {情报领域名称}
{领域整体态势分析，自然语言段落，关键数据加粗}

- [[{文件名}|{卡片标题}]] — {一句话简要说明}
- [[{文件名}|{卡片标题}]] — {一句话简要说明}

**商业模式影响**：{仅 Industry-Analysis、Vendor-Intelligence、Emerging-Tech 领域适用}

### {其他领域}
{同上结构，无情报的领域不显示}

## 重点关注

### {重点关注标题}
{深入分析描述，300-500 字，包含：背景上下文、技术细节、战略影响、关联情报综合分析}

**关键启示**：{给读者的启发性提示，2-3 句}

**关联情报**：[[{文件名}|{标题}]]、[[{文件名}|{标题}]]

### {其他重点关注}
{同上结构，3-4 条}

## 工作统计

| 指标 | 数量 |
|------|------|
| 新增情报卡片 | N |
| 情报领域覆盖 | N |
| 来源数量 | N |
| 重点关注情报 | N |

**领域分布**：{从 stats.domains_distribution 生成}
```

### 写作要求

- **情报概览**：先有领域聚合分析段落，再有情报卡片索引列表（带 Obsidian 链接）
- **重点关注**：筛选 3-4 条高价值情报，提供深入分析
- **客观性**：不添加主观解读，不进行趋势分析
- **篇幅**：根据情报量调整，不强制固定篇幅

## 周报设计

### 核心定位

**主题导向**：指导情报分析人员发现或关注特定的主题，从更大范围思考。

### 受众

执行层（安全团队、情报分析师）

### 输入来源

- 主输入：日报集合（本周 7 天）
- 回退：情报卡片（日报缺失时）
- 下钻：情报卡片（可选）

### 主题提炼方法

采用**语义分析**方案：
1. 读取本周 7 天日报全文
2. Agent 识别跨日报的模式和关联
3. 提炼主题并回溯相关日报
4. 主题数量动态调整（2-7 个，根据情报量）

### 内容结构

```markdown
# 情报周报 | YYYY-WXX

## 执行摘要
{本周主题概览，3-5 句话，概括本周涌现的核心主题}

## 主题分析

### 主题一：{主题标题}
{主题分析段落，综合本周多篇日报的相关内容，200-300 字}

**涉及变化类型**：{变化类型1}、{变化类型2}
{标注本主题涉及的变化类型，支撑月报按变化类型聚合}

**相关日报**：
- [[YYYY-MM-DD-daily|MM-DD 日报]] — {一句话说明}
- [[YYYY-MM-DD-daily|MM-DD 日报]] — {一句话说明}

**相关情报**：[[{文件名}|{标题}]]、[[{文件名}|{标题}]]

### 主题二：{主题标题}
{同上结构}

## 重点关注
{从各主题中筛选 3-4 个最值得关注的事件/趋势，提供深入分析}

### {重点关注标题}
{深入分析，300-500 字}

**关键启示**：{启发性提示}

## 下钻索引

### 本周日报

| 日期 | 标题 | 主题 |
|------|------|------|
| [[YYYY-MM-DD-daily|MM-DD]] | ... | {主题1}、{主题2} |

### 高价值情报卡片
{本周重点关注中涉及的情报卡片，按主题分组}

## 工作统计

| 指标 | 数量 |
|------|------|
| 本周情报卡片 | N |
| 日报覆盖 | 7/7 |
| 识别主题 | N |
| 重点关注 | N |

**主题分布**：{各主题涉及的情报数量}
```

### 写作要求

- **主题提炼**：通过语义分析发现跨日报的模式
- **主题数量**：动态调整，情报量少时 2-3 个，中等时 3-5 个，多时 5-7 个
- **变化类型标注**：每个主题必须标注涉及的变化类型（见月报变化类型定义），支撑月报聚合
- **关联呈现**：清晰展示主题与日报、情报卡片的关联
- **篇幅**：根据内容调整，不强制固定篇幅

## 月报设计

### 核心定位

**变化导向**：指导情报分析人员关注战略视角的各种变化，帮助发现机会和风险。

### 受众

决策层（CISO、安全总监）

### 输入来源

- 主输入：周报集合（本月 4-5 周）
- 下钻：日报（按需）
- 不访问情报卡片

### 变化类型定义

| 变化类型 | 覆盖领域 | 关注点 |
|----------|----------|--------|
| **新威胁出现** | Threat-Landscape | 新型攻击、威胁组织、漏洞趋势 |
| **市场格局变化** | Industry-Analysis, Vendor-Intelligence | 竞争格局、厂商战略、市场份额 |
| **技术突破与应用** | Emerging-Tech | 技术成熟、应用扩展、安全影响 |
| **客户需求演变** | Customer-Market | 预算调整、优先级转变、采购行为 |
| **合规压力变化** | Policy-Regulation | 新法规、合规要求、执法动态 |
| **资本动向** | Capital-Investment | 投资热度、投资方向、估值变化 |

### 内容结构

```markdown
# 情报月报 | YYYY-MM

## 执行摘要
{本月核心动态概览，3-5 句话}

## 情报动态

### 新态势

#### [S-YYYY-MM-XX] {态势标题} 🆕
{态势描述，200-300 字}

**变化类型**：{变化类型}

**代表事件**：
- [[YYYY-WXX-weekly|WXX]] — {一句话说明}

**关联情报**：[[{文件名}|{标题}]]

#### [S-YYYY-MM-XX] {态势标题} 🆕
{同上结构}

### 态势演进

#### [S-YYYY-MM-XX] {态势标题} 持续
{本月进展，200-300 字，说明与上月的变化}

**变化类型**：{变化类型}

**代表事件**：
- [[YYYY-WXX-weekly|WXX]] — {一句话说明}

#### [S-YYYY-MM-XX] {态势标题} ⬇️ 减弱
{本月进展，说明态势减弱的表现}

### 态势消退

#### [S-YYYY-MM-XX] {态势标题} ⏹️ 消退
{态势消退说明，简要描述为何不再值得关注}

## 态势观察

### 跨领域关联
{发现变化之间的关联关系}

### 模式识别
{从变化中发现的深层模式}

## 机会信号

{本月发现的机会信号，为年报的战略机会提供素材}

### 信号一：{信号标题}
{机会信号描述，当月发现的市场空白或机会点}

**相关态势**：[S-YYYY-MM-XX]

### 风险预警
{基于态势分析，识别的风险点}

### 行动建议
{给决策层的具体建议，2-3 条}

## 下钻索引

### 本月周报

| 周次 | 主题焦点 | 涉及变化类型 |
|------|----------|-------------|
| [[YYYY-WXX-weekly|WXX]] | ... | {变化类型1}、{变化类型2} |

### 日报链接（按变化类型）

**新威胁出现**：[[YYYY-MM-DD-daily]]、[[YYYY-MM-DD-daily]]

**市场格局变化**：[[YYYY-MM-DD-daily]]、[[YYYY-MM-DD-daily]]

## 态势索引

{本月所有态势的追踪状态汇总}

| 态势 ID | 标题 | 状态 | 首次出现 |
|---------|------|------|----------|
| S-2026-03-01 | AI 安全运营服务需求 | 持续 | 2026-03 |
| S-2026-04-01 | 供应链攻击工业化 | 🆕 新态势 | 2026-04 |

## 工作统计

| 指标 | 本月 |
|------|------|
| 情报卡片 | N |
| 周报覆盖 | 4/4 |
| 新态势 | N |
| 演进态势 | N |
| 消退态势 | N |
| 变化类型覆盖 | N/6 |

**变化类型分布**：{各变化类型的情报数量}
```

### 态势追踪机制

**态势 ID 规则**：
- 格式：`S-{年份}-{月份}-{序号}`
- 示例：`S-2026-04-01`
- 跨月追踪时 ID 保持不变

**态势状态定义**：

| 状态 | 符号 | 含义 | 判断标准 |
|------|------|------|----------|
| 新态势 | 🆕 | 本月首次出现 | 当月识别的态势 |
| 持续 | 无符号 | 保持关注，有新进展 | 有相关情报且热度不减 |
| 减弱 | ⬇️ | 热度下降 | 相关情报明显减少 |
| 消退 | ⏹️ | 不再值得关注 | 连续无相关情报或已解决 |

**年度聚合支撑**：
- 年报可基于态势 ID 追踪全年态势演变
- 态势索引提供月度状态快照
- 支撑年报的"趋势轨迹"分析

### 写作要求

- **动态章节**：有内容才显示，无内容则省略，避免空白章节
- **态势追踪**：每个态势必须有唯一 ID，跨月追踪时 ID 保持不变
- **状态标注**：新态势标 🆕，持续无符号，减弱标 ⬇️，消退标 ⏹️
- **机会信号**：月报发现的是"信号"而非"判断"，为年报的战略机会提供素材
- **跨领域视角**：按变化类型组织，发现领域间关联
- **叙述优先**：自然语言段落为主，列表仅用于辅助
- **篇幅**：根据内容调整，不强制固定篇幅

## 年报设计

### 核心定位

**趋势导向**：指导情报人员研究战略方向等重大命题。

回答三个核心问题：
1. 这一年行业朝什么方向演进？
2. 竞争格局如何变化？
3. 战略机会有哪些？

### 受众

战略层（高管、战略规划人员）

### 输入来源

- 主输入：月报集合 + 周报集合
- 下钻：日报（按需）
- 不访问情报卡片

### 内容结构

```markdown
# 情报年报 | YYYY

## 执行摘要
{年度核心判断，3-5 句话，回答：这一年网络安全行业最重要的变化是什么？}

## 年度趋势

### 趋势一：{趋势标题}

{开篇判断：一句话概括趋势的核心走向}

{方向判断段落}：
- 这一趋势的演进轨迹是什么？
- 推动趋势发展的核心动力是什么？
- 趋势背后的深层逻辑是什么？

{格局变化段落}：
- 哪些厂商在崛起？哪些在衰落？
- 市场集中度如何变化？
- 新进入者有哪些？他们的策略是什么？
- 中国市场与国际市场的差异？

{时机判断段落}：
- 技术处于什么成熟阶段？
- 市场接受度如何演变？
- 竞争窗口是否已关闭？还有哪些机会点？

**年度轨迹**：

| 季度 | 状态 | 关键进展 |
|------|------|----------|
| Q1 | 🆕 萌芽 | {概述} |
| Q2 | ⬆️ 加速 | {概述} |
| Q3 | ➡️ 稳定 | {概述} |
| Q4 | ⬇️ 回调 | {概述} |

**相关态势**：[S-YYYY-MM-XX]（{月报链接}）、[S-YYYY-MM-XX]（{月报链接}）

**支撑情报**：[[YYYY-MM-monthly|MM月月报]]、[[YYYY-WXX-weekly|WXX周报]]

### 趋势二：{趋势标题}
{同上结构}

## 战略机会

### 机会一：{机会标题}

**机会描述**：
{从情报分析中发现的市场空白}

**发现轨迹**：
{机会信号的月度演变，从哪个月的月报首次发现，后续如何验证}

**市场分析**：
{目标客户群体、市场规模判断、竞争态势}

**切入点分析**：
{可能的进入方式、所需能力资源}

**窗口判断**：
{机会窗口的持续时间、窗口关闭的触发条件}

**相关趋势**：{与哪个趋势关联，逻辑来源}

### 机会二：{机会标题}
{同上结构}

## 年度回顾

### 月报索引

| 月份 | 核心动向 | 涉及趋势 |
|------|----------|----------|
| [[YYYY-MM-monthly|MM月]] | ... | {趋势1}、{趋势2} |

### 态势全览

{全年态势的追踪状态汇总，按趋势分组}

| 态势 ID | 标题 | 首次出现 | 最终状态 | 涉及趋势 |
|---------|------|----------|----------|----------|
| S-2026-03-01 | AI 安全运营服务需求 | 2026-03 | 持续 | 趋势一 |
| S-2026-04-01 | 供应链攻击工业化 | 2026-04 | ⬇️ 减弱 | 趋势二 |

### 周报索引（按趋势）

**趋势一**：WXX、WXX、WXX...

**趋势二**：WXX、WXX...

## 年度统计

| 指标 | 本年 | 上年 | 同比 |
|------|------|------|------|
| 情报卡片 | N | N | +N% |
| 月报覆盖 | 12/12 | 12/12 | - |
| 识别趋势 | N | N | +N |
| 追踪态势 | N | N | +N |
| 战略机会 | N | N | - |

**变化类型分布**：{各变化类型的情报数量}
```

### 趋势状态定义

| 状态 | 符号 | 含义 |
|------|------|------|
| 年度最强趋势 | ⬆️⬆️ | 贯穿全年，影响深远 |
| 加速上升 | ⬆️ | 热度/影响持续增加 |
| 持续演变 | ➡️ | 保持关注，有新进展 |
| 趋于平稳 | ⏸️ | 已成常态，不再是热点 |
| 年末回落 | ⬇️ | 下半年热度下降 |

### 态势到趋势的聚合

年报生成时，需要将月报的态势聚合为年度趋势：

1. **识别相关态势**：查找同一变化类型、主题相关的态势
2. **追踪状态演变**：从态势索引中提取月度状态
3. **形成年度轨迹**：汇总为季度状态表格
4. **提炼趋势判断**：综合分析得出趋势结论

### 写作要求

- **叙述优先**：自然语言段落为主，列表仅用于辅助
- **逻辑可见**：因果关系、演进脉络清晰呈现
- **数据支撑**：数据嵌入叙述中，服务论点
- **结论先行**：每段开头给判断，再展开分析
- **动态感**：体现"从...到..."的演变过程
- **篇幅**：根据情报内容调整，有足够支撑就充分展开，不足就精简，核心是讲清楚但不废话

### 预计篇幅

| 章节 | 建议篇幅 |
|------|----------|
| 执行摘要 | 300-500 字 |
| 每个趋势 | 800-1200 字 |
| 每个战略机会 | 500-800 字 |
| 年度回顾 | 200-300 字 |

年度趋势预计 5-8 个，战略机会预计 3-5 个。

**全文预计 8000-12000 字**，根据实际情报内容调整。
