# 情报报告体系设计规范

> 本文档定义 market-radar 插件情报报告体系的设计规范，包括日报、周报、月报、年报的定位、结构和写作要求。

## 设计原则

### 层级聚合模型

报告体系采用**层级聚合**设计，高级别报告从低级别报告聚合生成，而非直接从情报卡片生成。

```
情报卡片
    ↓ 生成
日报（信号）
    ↓ 聚合 + 下钻
周报（主题）
    ↓ 聚合 + 下钻
月报（变化）
    ↓ 聚合 + 下钻
年报（趋势）
```

### 下钻设计

高级别报告支持下钻到低级别报告，但不一定需要直达情报卡片。

| 报告 | 主输入 | 按需下钻 | 是否访问情报卡片 |
|------|--------|----------|------------------|
| 日报 | 情报卡片 | - | ✅ 是 |
| 周报 | 日报集合 | 情报卡片 | ✅ 可选 |
| 月报 | 周报集合 | 日报 | ❌ 否 |
| 年报 | 月报 + 周报集合 | 日报 | ❌ 否 |

### 时间边界

| 报告 | 时间范围 |
|------|----------|
| 日报 | 当日（00:00-23:59） |
| 周报 | 周一至周日（ISO 周） |
| 月报 | 自然月（1日至月末） |
| 年报 | 自然年（1月1日至12月31日） |

## 日报设计

### 核心定位

**信号导向**：指导情报分析人员关注特定的情报。

### 受众

执行层（安全团队、情报分析师）

### 输入来源

情报卡片（按 `created_date` 过滤当日）

### 内容结构

```markdown
# 情报日报 | YYYY-MM-DD

## 执行摘要
{编辑点评视角，当日情报总体情况，3-5 句话}

## 情报概览

### {情报领域名称}
{领域整体态势分析，自然语言段落，关键数据加粗}

- [[{文件名}|{卡片标题}]] — {一句话简要说明}
- [[{文件名}|{卡片标题}]] — {一句话简要说明}

**商业模式影响**：{仅 Industry-Analysis、Vendor-Intelligence、Emerging-Tech 领域适用}

### {其他领域}
{同上结构，无情报的领域不显示}

## 重点关注

### {重点关注标题}
{深入分析描述，300-500 字，包含：背景上下文、技术细节、战略影响、关联情报综合分析}

**关键启示**：{给读者的启发性提示，2-3 句}

**关联情报**：[[{文件名}|{标题}]]、[[{文件名}|{标题}]]

### {其他重点关注}
{同上结构，3-4 条}

## 工作统计

| 指标 | 数量 |
|------|------|
| 新增情报卡片 | N |
| 情报领域覆盖 | N |
| 来源数量 | N |
| 重点关注情报 | N |

**领域分布**：{从 stats.domains_distribution 生成}
```

### 写作要求

- **情报概览**：先有领域聚合分析段落，再有情报卡片索引列表（带 Obsidian 链接）
- **重点关注**：筛选 3-4 条高价值情报，提供深入分析
- **客观性**：不添加主观解读，不进行趋势分析
- **篇幅**：根据情报量调整，不强制固定篇幅

## 周报设计

### 核心定位

**主题导向**：指导情报分析人员发现或关注特定的主题，从更大范围思考。

### 受众

执行层（安全团队、情报分析师）

### 输入来源

- 主输入：日报集合（本周 7 天）
- 回退：情报卡片（日报缺失时）
- 下钻：情报卡片（可选）

### 主题提炼方法

采用**语义分析**方案：
1. 读取本周 7 天日报全文
2. Agent 识别跨日报的模式和关联
3. 提炼主题并回溯相关日报
4. 主题数量动态调整（2-7 个，根据情报量）

### 内容结构

```markdown
# 情报周报 | YYYY-WXX

## 执行摘要
{本周主题概览，3-5 句话，概括本周涌现的核心主题}

## 主题分析

### 主题一：{主题标题}
{主题分析段落，综合本周多篇日报的相关内容，200-300 字}

**相关日报**：
- [[YYYY-MM-DD-daily|MM-DD 日报]] — {一句话说明}
- [[YYYY-MM-DD-daily|MM-DD 日报]] — {一句话说明}

**相关情报**：[[{文件名}|{标题}]]、[[{文件名}|{标题}]]

### 主题二：{主题标题}
{同上结构}

## 重点关注
{从各主题中筛选 3-4 个最值得关注的事件/趋势，提供深入分析}

### {重点关注标题}
{深入分析，300-500 字}

**关键启示**：{启发性提示}

## 下钻索引

### 本周日报

| 日期 | 标题 | 主题 |
|------|------|------|
| [[YYYY-MM-DD-daily|MM-DD]] | ... | {主题1}、{主题2} |

### 高价值情报卡片
{本周重点关注中涉及的情报卡片，按主题分组}

## 工作统计

| 指标 | 数量 |
|------|------|
| 本周情报卡片 | N |
| 日报覆盖 | 7/7 |
| 识别主题 | N |
| 重点关注 | N |

**主题分布**：{各主题涉及的情报数量}
```

### 写作要求

- **主题提炼**：通过语义分析发现跨日报的模式
- **主题数量**：动态调整，情报量少时 2-3 个，中等时 3-5 个，多时 5-7 个
- **关联呈现**：清晰展示主题与日报、情报卡片的关联
- **篇幅**：根据内容调整，不强制固定篇幅

## 月报设计

### 核心定位

**变化导向**：指导情报分析人员关注战略视角的各种变化，帮助发现机会和风险。

### 受众

决策层（CISO、安全总监）

### 输入来源

- 主输入：周报集合（本月 4-5 周）
- 下钻：日报（按需）
- 不访问情报卡片

### 变化类型定义

| 变化类型 | 覆盖领域 | 关注点 |
|----------|----------|--------|
| **新威胁出现** | Threat-Landscape | 新型攻击、威胁组织、漏洞趋势 |
| **市场格局变化** | Industry-Analysis, Vendor-Intelligence | 竞争格局、厂商战略、市场份额 |
| **技术突破与应用** | Emerging-Tech | 技术成熟、应用扩展、安全影响 |
| **客户需求演变** | Customer-Market | 预算调整、优先级转变、采购行为 |
| **合规压力变化** | Policy-Regulation | 新法规、合规要求、执法动态 |
| **资本动向** | Capital-Investment | 投资热度、投资方向、估值变化 |

### 内容结构

```markdown
# 情报月报 | YYYY-MM

## 执行摘要
{本月核心动态概览，3-5 句话}

## 情报动态

### 新态势
{本月新出现的态势，每个态势一段描述，跨领域视角}

**代表事件**：
- [[YYYY-WXX-weekly|WXX]] — {一句话说明}

**关联情报**：[[{文件名}|{标题}]]

### 态势演进
{已关注态势的本月进展}

**代表事件**：
- [[YYYY-WXX-weekly|WXX]] — {一句话说明}

## 态势观察

### 跨领域关联
{发现变化之间的关联关系}

### 模式识别
{从变化中发现的深层模式}

## 战略动向

### 机会识别
{基于态势分析，识别的市场机会、技术方向}

### 风险预警
{基于态势分析，识别的风险点}

### 行动建议
{给决策层的具体建议，2-3 条}

## 下钻索引

### 本月周报

| 周次 | 主题焦点 | 涉及变化类型 |
|------|----------|-------------|
| [[YYYY-WXX-weekly|WXX]] | ... | {变化类型1}、{变化类型2} |

### 日报链接（按变化类型）

**新威胁出现**：[[YYYY-MM-DD-daily]]、[[YYYY-MM-DD-daily]]

**市场格局变化**：[[YYYY-MM-DD-daily]]、[[YYYY-MM-DD-daily]]

## 工作统计

| 指标 | 本月 |
|------|------|
| 情报卡片 | N |
| 周报覆盖 | 4/4 |
| 变化类型覆盖 | N/6 |

**变化类型分布**：{各变化类型的情报数量}
```

### 写作要求

- **动态章节**：有内容才显示，无内容则省略，避免空白章节
- **跨领域视角**：按变化类型组织，发现领域间关联
- **叙述优先**：自然语言段落为主，列表仅用于辅助
- **篇幅**：根据内容调整，不强制固定篇幅

## 年报设计

### 核心定位

**趋势导向**：指导情报人员研究战略方向等重大命题。

回答三个核心问题：
1. 这一年行业朝什么方向演进？
2. 竞争格局如何变化？
3. 战略机会有哪些？

### 受众

战略层（高管、战略规划人员）

### 输入来源

- 主输入：月报集合 + 周报集合
- 下钻：日报（按需）
- 不访问情报卡片

### 内容结构

```markdown
# 情报年报 | YYYY

## 执行摘要
{年度核心判断，3-5 句话，回答：这一年网络安全行业最重要的变化是什么？}

## 年度趋势

### 趋势一：{趋势标题}

{开篇判断：一句话概括趋势的核心走向}

{方向判断段落}：
- 这一趋势的演进轨迹是什么？
- 推动趋势发展的核心动力是什么？
- 趋势背后的深层逻辑是什么？

{格局变化段落}：
- 哪些厂商在崛起？哪些在衰落？
- 市场集中度如何变化？
- 新进入者有哪些？他们的策略是什么？
- 中国市场与国际市场的差异？

{时机判断段落}：
- 技术处于什么成熟阶段？
- 市场接受度如何演变？
- 竞争窗口是否已关闭？还有哪些机会点？

**支撑情报**：[[YYYY-MM-monthly|MM月月报]]、[[YYYY-WXX-weekly|WXX周报]]

### 趋势二：{趋势标题}
{同上结构}

## 战略机会

### 机会一：{机会标题}

**机会描述**：
{从情报分析中发现的市场空白}

**市场分析**：
{目标客户群体、市场规模判断、竞争态势}

**切入点分析**：
{可能的进入方式、所需能力资源}

**窗口判断**：
{机会窗口的持续时间、窗口关闭的触发条件}

**相关趋势**：{与哪个趋势关联，逻辑来源}

### 机会二：{机会标题}
{同上结构}

## 年度回顾

### 月报索引

| 月份 | 核心动向 | 涉及趋势 |
|------|----------|----------|
| [[YYYY-MM-monthly|MM月]] | ... | {趋势1}、{趋势2} |

### 周报索引（按趋势）

**趋势一**：WXX、WXX、WXX...

**趋势二**：WXX、WXX...

## 年度统计

| 指标 | 本年 | 上年 | 同比 |
|------|------|------|------|
| 情报卡片 | N | N | +N% |
| 月报覆盖 | 12/12 | 12/12 | - |
| 识别趋势 | N | N | +N |
| 战略机会 | N | N | - |

**变化类型分布**：{各变化类型的情报数量}
```

### 写作要求

- **叙述优先**：自然语言段落为主，列表仅用于辅助
- **逻辑可见**：因果关系、演进脉络清晰呈现
- **数据支撑**：数据嵌入叙述中，服务论点
- **结论先行**：每段开头给判断，再展开分析
- **动态感**：体现"从...到..."的演变过程
- **篇幅**：根据情报内容调整，有足够支撑就充分展开，不足就精简，核心是讲清楚但不废话

### 预计篇幅

| 章节 | 建议篇幅 |
|------|----------|
| 执行摘要 | 300-500 字 |
| 每个趋势 | 800-1200 字 |
| 每个战略机会 | 500-800 字 |
| 年度回顾 | 200-300 字 |

年度趋势预计 5-8 个，战略机会预计 3-5 个。

**全文预计 8000-12000 字**，根据实际情报内容调整。
